# Resumen: Requerimientos del Software

Los requerimientos de software son las necesidades, objetivos y restricciones que debe cumplir un sistema para satisfacer a los usuarios y al negocio.

Se clasifican en:

**Requerimientos funcionales:** describen las funciones que debe realizar el sistema, como gestionar usuarios, registrar datos, realizar consultas y generar reportes.
**Requerimientos no funcionales:** establecen cómo debe funcionar el sistema, incluyendo aspectos como seguridad, rendimiento, usabilidad, disponibilidad y confiabilidad.

Para obtener los requerimientos se utilizan técnicas como entrevistas, encuestas, observación y reuniones con los interesados. Posteriormente, se documentan mediante historias de usuario, casos de uso, especificaciones y prototipos.

Finalmente, los requerimientos se validan con los clientes mediante revisiones y pruebas de aceptación para asegurar que el sistema cumpla las necesidades planteadas. Todo este proceso ayuda a reducir errores, mejorar la calidad del software y facilitar su diseño y desarrollo.
